Use a SQL Server Reporting Services report view
See Also






The SQL Server Reporting Services report view uses SQL Server Reporting Services to provide interactive reports that can be displayed in a wide variety of formats.

If your organization uses SQL Server Reporting Services reports and has enabled their use in Microsoft Office Business Scorecard Manager 2005, you can view these reports in the Office Report View Web Part.

To use a SQL Server Reporting Services report view

  • In Office Report View, do any of the following:

    • To navigate within a reporting services report:

      On the Reporting Services toolbar, click Show or Hide DocMap to open the DocMap list. Then select an entry in the list. Each entry in the DocMap list corresponds to a section of the report.

    • To navigate between pages in a SQL Server Reporting Services report:

      On the Reporting Services toolbar, click the forward and back arrows.

    • To increase or decrease the visual display size of the SQL Server Reporting Services report:

      On the Reporting Services toolbar, click the arrow next to the Zoom box, and then click the zoom setting that you want to use.

    • To search the SQL Server Reporting Services report for a specific word or number:

      In the Find text box, type the word or number, and then click Find to locate the first instance of the word or number in the report. Click Next to find the next instance of the word or number in the report.

    • To export the SQL Server Reporting Services report for use in another application:

      On the Reporting Services toolbar, select a file format in the Select a format list, and then click Export.

    • To refresh the SQL Server Reporting Services report:

      On the Reporting Services toolbar, click Refresh.

    • To print a SQL Server Reporting Services report:

      On the Reporting Services toolbar, click Print.

    • To read online help for SQL Server Reporting Services:

      On the Reporting Services toolbar, click Help.
