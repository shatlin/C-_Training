Scorecard security
See Also






You can help make Microsoft Office Business Scorecard Manager 2005 more secure by using the security features of SharePoint Products and Technologies and Microsoft Office Business Scorecard Builder.

We recommend that you use Secure Sockets Layer (SSL) or some other form of encryption for any Web site that hosts Business Scorecard Manager data. Encryption helps to secure the data that is transmitted every time a user accesses a Microsoft Office Business Scorecard Manager Server Web Part from being intercepted by unauthorized users. For more information about securing your Business Scorecard Manager data, see the Microsoft® Office Business Scorecard Manager 2005 Planning and Administration Guide.

Note:
Under the default security settings, any user who has access to a Windows SharePoint Services-based or SharePoint Portal Server-based Web site that hosts a Microsoft Office Business Scorecard Manager Server Web Part might potentially have access to any business data that the scorecard displays, even if the user does not have permission to access the data source directly. The extra protection provided by the security features of SharePoint Products and Technologies restricts this access.
